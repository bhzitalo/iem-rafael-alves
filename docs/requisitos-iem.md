# 📄 Análise de Requisitos
Este documento descreve os requisitos do sistema desenvolvido para o professor de música Rafael Alves. O objetivo do site é apresentar seus serviços, cursos disponíveis e facilitar o contato com possíveis alunos.

[Ver site na web](https://iem-rafaelalves.netlify.app/)

## 🎯 2. Objetivo do Sistema
Desenvolver um site institucional simples que permita:
- Apresentar o professor
- Divulgar cursos de música
- Facilitar o contato com interessados

## 👥 3. Público-Alvo
- Jovens iniciantes na música
- Pessoas interessadas em aprender violão e teoria musical
- Usuários que buscam aulas acessíveis e objetivas

## ⚙️ 4. Requisitos Funcionais

- **RF01:** O sistema deve exibir uma página inicial com apresentação do professor  
- **RF02:** O sistema deve conter uma página “Sobre” com informações do professor  
- **RF03:** O sistema deve exibir uma página de cursos disponíveis  
- **RF04:** O sistema deve conter uma página de contato  
- **RF05:** O sistema deve permitir navegação entre as páginas através de um menu  

## 🛠️ 5. Requisitos Não Funcionais

- **RNF01:** O site deve ser desenvolvido em HTML  
- **RNF02:** O site deve ser acessível em dispositivos móveis (responsivo)  
- **RNF03:** O conteúdo deve ser claro e organizado  
- **RNF04:** As páginas devem possuir títulos definidos  
- **RNF05:** O sistema deve carregar rapidamente  

## 📑 6. Estrutura do Sistema

O site será composto por 4 páginas principais:

- **index.html** → Página inicial  
- **sobre.html** → Informações do professor  
- **cursos.html** → Lista de cursos  
- **contato.html** → Formulário e informações de contato  

## ⚠️ 7. Observação

O site será desenvolvido propositalmente com alguns problemas de conteúdo e estrutura, com o objetivo de permitir análise e correção por outro grupo, conforme solicitado na atividade avaliativa.
